# Mapa Rápido de Referência

Base local: `content/system-prompts-and-models-of-ai-tools`

## Fontes prioritárias

- `Cursor Prompts/`
  - Quando usar: comportamento de agente de código, execução com ferramentas, padrões de resposta.
- `v0 Prompts and Tools/`
  - Quando usar: ideação de UI, estruturação de entregas visuais e componentes.
- `Lovable/`
  - Quando usar: estilo de construção de páginas e experiência de produto.
- `VSCode Agent/`
  - Quando usar: automação de tarefas de desenvolvimento e decisões de execução.
- `Windsurf/` e `Trae/`
  - Quando usar: comparação de estratégia entre agentes e fluxos alternativos.

## Fontes secundárias

- `Augment Code/`, `CodeBuddy Prompts/`, `Comet Assistant/`, `Replit/`, `Perplexity/`, `Open Source prompts/`
  - Use para ampliar cobertura quando as fontes prioritárias não forem suficientes.

## Checklist de extração

Antes de aplicar qualquer referência, responder:

1. Qual problema real do usuário isso resolve?
2. A regra é executável no projeto atual (React + TypeScript + Vite)?
3. A referência melhora clareza, qualidade ou velocidade da entrega?
4. Há conflito com convenções existentes do repositório?

## Formato recomendado de aplicação

- **Contexto da tarefa**: objetivo + restrições.
- **Padrões escolhidos**: lista curta de regras acionáveis.
- **Aplicação no código**: o que foi alterado e por quê.
- **Validação**: como testar/verificar.
